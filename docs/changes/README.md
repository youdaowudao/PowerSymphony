# 单次变更文档

本目录用于单次高风险变更的设计、计划、验证文档。

入口顺序：

1. 先看本目录 `README.md`
2. 再看当前 change 的入口文件
3. 只读该入口文件明确点名的 `SPEC/` 或 `阶段规划/` 文档

文件数量和拆分深度不设上限，由作者按任务复杂度自行决定。

最低要求：

- 有一个清晰入口文件，能说明这次 change 的目标和文档索引。
- 每个文件职责清楚，不重复抄写同一内容。
- 文件命名便于排序、检索和 handoff。
- 文档里必须有一个稳定的“目标 / 需求快照”段落。

常见但非强制的结构：

```text
docs/changes/<change-id>/
  00_context.md
  10_design.md
  20_plan.md
  90_verification.md
```

小修小改默认不新建 repo 文档。

## “目标 / 需求快照”最低内容

为了让 reviewer 尽量不依赖 Linear，本目录的入口文件应至少回答：

- 人类原本希望解决什么问题
- 这次 change 的成功标准是什么
- 这次明确不做什么
- 当前已知的固定约束或风险是什么

这里写的是稳定结论，不是把 Linear 整段对话原样复制进来。
