# 时间线

## 关键演进

- 先出现了“旧 owner 还在跑，但本地 gate 已经被释放”的风险窗口。
- 随后同一 issue 在 stall / stop 路径下可能被再次派发。
- 再往后，运行摘要和快照开始出现跨代字段混拼的风险。
- 这次事故分析最终把问题收敛为：ownership gate 的释放时机过早，且没有足够强的 generation 边界。

## 这次时间线想证明什么

不是证明“代码哪里长错了”，而是证明：

1. 问题不是偶发 UI 假象。
2. 问题不是单独某个 worker 的局部错误。
3. 问题是 gate、stop、summary 三条链路组合后的系统性缺口。
