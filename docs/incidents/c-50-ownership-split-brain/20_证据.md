# 证据

## 证据类型

- orchestrator 的 dispatch / release 顺序
- `terminate_running_issue/3` 与 `should_dispatch_issue?/6` 的 gate 行为
- `AppServer.stop_session/1` 只关本地 port，不具备完整远端收口协议
- `RunStateStore` 与 snapshot 的跨代混拼路径
- `docs/changes/c-50-ownership-gate/README.md` 中的稳定目标快照

## 证据结论

- 旧 owner 未被安全确认结束时，系统仍可能重派发。
- 现有 summary 机制不足以防止跨代字段混入同一行展示。
- 仅靠“看见某个 worker 已经开始新一轮”不能反证旧 owner 已经安全死亡。
