# 阶段规划入口

本目录用于维护当前长期主题在各阶段的正式规划、阶段门禁、冻结边界和交接说明。

这里回答的问题是：

- 当前阶段做什么
- 当前阶段明确不做什么
- 当前阶段的完成标准是什么
- 当前阶段有哪些冻结结论和交接条件

这里不负责：

- 长期主题的最终愿景
- 所有阶段的总路线图
- 长期有效的专题 SPEC 正文

当前阶段文档：

- [阶段1_控制面骨架与门禁.md](./阶段1_控制面骨架与门禁.md)
- [阶段4_运行追踪与状态推导门禁.md](./阶段4_运行追踪与状态推导门禁.md)

写作者路由：

- 阶段目标、阶段范围、阶段门禁、阶段 handoff：
  - 写在本目录
- 如果只是调整长期路线图，不进入某个阶段的具体交付边界：
  - 改到 `长期规划/20_路线图与里程碑.md`
- 如果要定义长期有效的架构或质量 SPEC：
  - 改到上一级的 `SPEC/`
