# 长期主题文档入口

本目录用于长期愿景、阶段路线、里程碑、未完成功能清单、技术路线分析、A/B 裁决，以及供 coder/reviewer 直接引用的长期 SPEC。

适合放在这里的内容：

- 一个项目未来 3 到 12 个月的目标
- 分阶段的路线图
- 暂未完成但需要长期追踪的功能
- 与单次 change 无法绑定的持续性 backlog
- 某个长期主题内部的技术路线选择
- 同一长期主题下的 A/B 方案裁决
- 因需求变更而需要重写的阶段目标或 backlog 优先级

推荐结构：

```text
docs/initiatives/
  README.md
  全局规划/
  阶段规划/
  SPEC/
  决策记录/
  资料复核/
```

补充规则：

- 单次实现细节仍优先写到 `docs/changes/`。
- 如果某次需求变更已经改变了长期愿景、阶段路线或 backlog 排序，再同步更新 `全局规划/`。
- `README.md` 负责回答“谁该看哪里、谁该写哪里”。
- `全局规划/` 放长期目标、路线图、里程碑。
- `阶段规划/` 放阶段级正式规划、阶段门禁、冻结边界和交接要求。
- `SPEC/` 放长期有效、可被 coder 和 reviewer 直接引用的专题 SPEC。
- `决策记录/` 放关键 A/B 裁决与路线冻结结论。
- `资料复核/` 放资料来源、源码核对与复核记录。

## 写作者路由

- 写长期目标、成功标准、路线图：
  - 去 `全局规划/`
- 写阶段范围、阶段门禁、阶段交接：
  - 去 `阶段规划/`
- 写长期架构、观测、Workflow、质量门禁 SPEC：
  - 去 `SPEC/`
- 写路线裁决：
  - 去 `决策记录/`
- 写资料来源与源码复核：
  - 去 `资料复核/`

## coder / reviewer 路由

coder 和 reviewer 不需要通读整个目录，默认路径是：

1. 先看 `AGENTS.md`
2. 再看当前 `docs/changes/<change-id>/README.md`
3. 只读取 change README 明确点名的 `SPEC/` 或 `阶段规划/` 文档
