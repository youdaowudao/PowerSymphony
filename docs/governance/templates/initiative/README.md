# 长期主题入口模板

建议结构：

```text
docs/initiatives/
  README.md
  全局规划/
    10_愿景与成功标准.md
    20_路线图与里程碑.md
  阶段规划/
    阶段1_示例阶段规划.md
  SPEC/
    30_长期_SPEC.md
    33_质量门禁与验收边界.md
  决策记录/
    2026-05-03_路线裁决.md
  资料复核/
    2026-05-03_资料来源与源码复核.md
```

说明：

- `长期规划/` 放长期目标、路线图和里程碑。
- `阶段规划/` 放阶段级正式规划、阶段门禁、冻结边界和交接要求。
- `SPEC/` 放长期有效、可被 coder 直接引用的 SPEC。
- `决策记录/` 放关键 A/B 裁决和路线冻结结论。
- `资料复核/` 放资料来源、源码核对和复核记录。
- 如果长期主题较小，可以减少文件数量，但仍建议保留这些用途分层。

建议至少回答这些问题：

- 长期想达到什么结果
- 为什么现在要做这件事
- 当前分几个阶段推进
- 每个阶段的完成标准是什么
- 还有哪些未完成功能和 backlog
- 当前技术路线是什么
- 哪些关键 A/B 裁决已经做出
